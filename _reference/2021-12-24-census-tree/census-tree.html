The Census collects a lot of information. The Bureau uses variables to identify the information they distribute digitally. Related variables are usually numbered close together, but not always. Knowing which variable you‚Äôre looking for can make research go faster, and ensures you‚Äôre working with the right data.1 A list of all variables is available on the Census API website. The tidycensus R package makes this information easily available so you can search and filter for the variables you need.

Backstory

I looked into how to make diagrams using DiagrammeR, but I couldn‚Äôt figure out the functions to turn the variables into a massive diagram.

R Code

Without further ado, here‚Äôs how to reorganize Census variables to create a folder tree.

Get variables

To get started, we‚Äôll need the tidycensus package to access the Census API‚Äôs list of variables, and tidyverse to wrangle the data into the correct format.

library(tidyverse)
library(tidycensus)

Use the load_variables() function to get the variables from the dataset you need. I also went ahead and narrowed for housing-related variables (which start with ‚ÄúB25‚Ä¶‚Äù).

v19 <- load_variables(2019, "acs5")

hsg <- v19 %>% 
  filter(str_detect(name, "^B25"))

We have the information we need to create our directory list.

Tidy

Now we need to reorganize and clean the data into one column.

filter(): There are 2,207 variables related to housing, but I don‚Äôt want a tree that is too long to digest (I can always make more trees). But every single age bracket, income bracket, etc. has its own variable, repeated many times over. So I narrowed to only the top-level variables, ending with ‚Äú_001‚Äù. Some variables are also available by race or as aggregates or medians. These variables are useful but will not be the focus of this tree.

mutate(): Census data is pretty standardized, so there‚Äôs not a lot of work to change the variables into what we need. Most of the mutating is related to cleaning up punctuation and spaces, since we don‚Äôt really want those in a file name. I omitted the ‚ÄúB‚Äù at the start of every variable, so the variable ID number will appear at the top of folder lists.

Every time ‚Äù BY ‚Äù is mentioned in the concept column, it indicates a subvariable, which we need to turn into a subfolder /. Then append the name column to include the variable ID using unite().

h1 <- hsg %>% 
  filter(str_detect(name, "_001$"),           #' only top-level variables
         !str_detect(name, "[:alpha:]_"),     #' omit race variables
         str_detect(label, "Estimate!!Total") #' omit aggregates, medians, etc.
         ) %>% 
  select(-label) %>%     #' done using label column
  mutate(concept = str_replace_all(concept, "-(?=[:alpha:])", " "),
         concept = str_remove_all(concept, "[:punct:]"),
         concept = str_replace_all(concept, " BY ", "/"),
         name = str_remove(name, "_001"),
         name = str_remove_all(name, "^B")
         ) %>% 
  unite(col = "fileStr", concept, name, sep = "/", na.rm = TRUE) %>% 
  mutate(fileStr = str_replace_all(fileStr, "\\s", "-"),
         fileStr = str_replace_all(fileStr, "-{2,}", "-"))

Export

Turn the column into a list of values. Then use write_lines() to create a .txt üìÑ, writing one line for every value listed.

vH1 <- h1$fileStr

write_lines(vH1, file = "~/Documents/H1.txt")

This outputs the .txt üìÑ to your Documents üìÅ.

Terminal

Open the Terminal. Create a new üìÅ to contain the tree and cd into the new üìÅ.

mkdir ~/Documents/census
cd Documents/census

Below is the function that creates a new üìÅ for every row in the .txt üìÑ.

xargs -I {} mkdir -p "{}" < "../H1.txt"
